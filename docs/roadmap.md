# SUDOMG GhostNote Roadmap

SUDOMG GhostNote is being built in stages. Each release expands on the previous one while staying true to a single goal:

**Help people remember meaningful work.**

This roadmap focuses on long-term product direction rather than day-to-day implementation details.

---

# Version 1.0 — Capture my work.

**Theme**

Provide the fastest, simplest way to capture meaningful work before it's forgotten.

The first public release establishes SUDOMG GhostNote as a polished, local-first capture tool.

## Goals

* Fast note capture
* Local SQLite storage
* Search
* Date filtering
* Categories
* Edit existing entries
* Dark & Light themes
* GhostyQuotes
* Unified Settings window
* CSV & Markdown export
* Right-click context menu
* Standalone Windows executable
* Windows installer
* AppData migration
* Documentation and release preparation

---

# Version 1.5 — Help me remember to capture my work.

**Theme**

Reduce forgotten work by reminding users at the right time—without becoming intrusive or feeling like productivity monitoring.

SUDOMG GhostNote should fit naturally into a user's workflow and make capturing work almost effortless.

## Planned Features

* Background service
* Scheduled reminders
* Work hours
* Idle-based prompting
* System tray integration
* Global hotkey
* Better export options
* Additional viewer polish

The goal is not to automate note-taking—it is to help users remember the work only they can accurately describe.

---

# Version 2.0+ — Help me understand the work I've already captured.

**Theme**

Turn captured work into meaningful insights.

By this point, users will have built a history of Notes. SUDOMG GhostNote should help transform that history into useful summaries, reports, and evidence without replacing the user's judgment.

## Planned Features

* Weekly AI summaries (Echoes)
* Monthly AI summaries
* AI-assisted reports
* Work graphs and trends
* Promotion evidence generation
* Resume-friendly exports
* Signals (high-impact work detection)
* Meeting summaries
* Context-aware suggestions

AI should always assist the user—not automatically create work records.

---

# Future Integrations

Potential future input and output integrations include:

* Slack
* Email
* Calendar
* Zoom
* GitHub / Git
* Linear

Integrations should always be:

* Optional
* Transparent
* User-controlled

Their purpose is to improve recall and reporting—not monitor productivity.

---

# Guiding Principles

Every release should improve one or more of the following:

* Capture speed
* Recall quality
* User ownership
* Viewer usability

without compromising:

* Privacy
* Transparency
* Simplicity
* Local-first operation

SUDOMG GhostNote exists to help users remember meaningful work—not to measure productivity.

---

# Parking Lot

Ideas worth exploring in future releases:

* Screenshot attachments
* Meeting templates
* Timeline improvements
* Better category management
* Additional export formats
* Custom reminder rules
* Custom report templates

These ideas are intentionally uncommitted and may evolve over time.

---

# Maintaining This Roadmap

This roadmap should only change when:

* A major release ships
* Product priorities change
* A significant architectural decision is made
* A new long-term direction is established

Daily tasks belong in GitHub Issues, TODOs, or personal notes.

This roadmap exists to preserve the vision of SUDOMG GhostNote—not to track every implementation detail.
