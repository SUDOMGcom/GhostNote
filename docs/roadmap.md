# SUDOMG GhostNote Roadmap

GhostNote is being built in stages.

Each release expands on the previous one while staying true to a single goal:

> **Help people remember meaningful work.**

This roadmap documents long-term product direction rather than day-to-day implementation tasks.

---

# Version 1 — Capture my work.

## Theme

Provide the fastest, simplest way to capture meaningful work before it's forgotten.

Version 1 establishes GhostNote as a polished, local-first Windows capture utility.

## Included

- Fast GhostNote capture
- Local SQLite database
- Search
- Date filtering
- Categories
- Editing
- GhostyQuotes
- Unified Settings
- CSV export
- Markdown export
- Right-click desktop integration
- Windows installer
- Local AppData storage

---

# Version 1.5 — Help me remember to capture my work.

## Theme

Reduce forgotten work without becoming intrusive or feeling like productivity monitoring.

GhostNote should fit naturally into a user's workflow and make capture almost effortless.

## Planned Features

- Background service
- Scheduled reminders
- Work hours
- Idle-based prompting
- System tray integration
- Global hotkey
- Viewer polish
- Better exports

The goal is not to automate note-taking.

The goal is to help users remember work only they can accurately describe.

---

# Version 2+ — Help me understand the work I've already captured.

## Theme

Transform captured work into meaningful insights.

Users will have built a history of GhostNotes.

GhostNote should help turn those notes into useful summaries and reports without replacing the user's judgment.

## Planned Features

- Weekly Echoes
- Monthly Echoes
- AI-assisted reports
- Signals
- Work graphs
- Promotion evidence
- Resume exports
- Meeting summaries
- Context-aware suggestions

AI should always assist the user—not automatically create work records.

---

# Future Integrations

Potential integrations include:

- Slack
- Email
- Calendar
- Zoom
- GitHub / Git
- Linear

Integrations should always be:

- Optional
- Transparent
- User-controlled

Their purpose is improving recall—not monitoring productivity.

---

# Monetization

Capture should remain the heart of GhostNote.

The free version should always remain genuinely useful.

Future paid features will likely focus on:

- AI
- Integrations
- Advanced reporting
- Promotion evidence
- Resume generation

Paid features should save time—not lock away core functionality.

---

# Guiding Principles

Every release should improve one or more of:

- Capture speed
- Recall quality
- Viewer usability
- User ownership

without compromising:

- Privacy
- Transparency
- Simplicity
- Local-first operation

GhostNote exists to help users remember meaningful work—not measure productivity.

---

# Parking Lot

Ideas worth exploring:

- Screenshot attachments
- Meeting templates
- Timeline improvements
- Better category management
- Additional export formats
- Custom reminder rules
- Custom report templates

These ideas are intentionally uncommitted.

---

# Maintaining This Roadmap

Update this roadmap only when:

- A major version ships.
- Product priorities change.
- A significant architectural decision is made.
- Long-term direction changes.

Daily implementation belongs in GitHub Issues, project notes, or TODOs.

This roadmap exists to preserve GhostNote's vision—not to track every task.
